# 产品需求文档 (PRD)

## 文档元信息

| 项目 | 内容 |
|------|------|
| 产品名称 | 智能旅行助手 (AI Travel Planner) |
| 版本 | v1.0 |
| 目标用户 | 自由行旅行者、不擅长做攻略的用户 |
| 核心价值 | 5 分钟内生成个性化、可编辑、可导出的完整旅行计划 |

---

## 1. 背景与问题

### 1.1 用户痛点

| 痛点 | 描述 | 严重程度 |
|------|------|----------|
| 信息分散 | 用户需要在多个平台查询景点、酒店、天气、攻略 | 高 |
| 缺少个性化 | 通用攻略无法匹配个人偏好和预算 | 高 |
| 难以调整 | 计划修改后需要重新查找信息 | 中 |
| 预算失控 | 旅行费用难以预估 | 中 |
| 无地图感 | 不了解景点之间的地理位置关系 | 低 |

### 1.2 解决方案

通过 AI Agent 自动搜索、整合、规划，用户只需输入基本信息即可获得：
- 结构化的每日行程
- 基于地图的景点分布
- 透明的预算明细
- 可编辑、可导出的旅行计划

---

## 2. 目标用户

### 2.1 用户画像

| 画像 | 特征 | 场景 |
|------|------|------|
| **自由行新手** | 没有旅行规划经验，不知道从哪开始 | "我想去北京玩3天，有什么推荐？" |
| **效率型旅客** | 时间紧张，不想花大量时间做攻略 | "下周末去上海，帮我快速规划" |
| **家庭出游者** | 需要考虑老人/小孩，对预算敏感 | "带老人去成都5天，预算1万" |
| **深度体验者** | 追求个性化体验，讨厌大众路线 | "杭州小众文艺路线3天" |

## 3. 功能需求

### 3.1 功能清单

| 编号 | 功能 | 优先级 | 描述 |
|------|------|--------|------|
| F1 | 行程生成 | P0 | 输入目的地、日期、偏好、预算，生成完整行程 |
| F2 | 地图可视化 | P0 | 在地图上标注所有景点位置 |
| F3 | 预算明细 | P0 | 展示门票/酒店/餐饮/交通分项和总计 |
| F4 | 天气信息 | P1 | 展示行程期间的每日天气 |
| F5 | 行程编辑 | P1 | 支持拖拽调整景点顺序、添加/删除景点 |
| F6 | 导出 PDF | P1 | 将行程导出为 PDF 文件 |
| F7 | 导出图片 | P2 | 将行程导出为图片 |
| F8 | 进度提示 | P2 | 生成行程时展示进度条和状态 |
| F9 | 偏好模板 | P2 | 提供常用偏好模板（历史文化/自然风光/美食购物等） |
| F10 | 多方案对比 | P3 | 同一目的地生成多个方案供选择 |

### 3.2 功能详述

#### F1: 行程生成（核心功能）

**输入项**：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 目的地城市 | 文本 | 是 | — | 支持中文城市名 |
| 开始日期 | 日期 | 是 | — | YYYY-MM-DD |
| 结束日期 | 日期 | 是 | — | 需 >= 开始日期 |
| 偏好 | 单选 | 否 | 历史文化 | 历史文化/自然风光/美食购物/亲子游玩/文艺打卡 |
| 预算水平 | 单选 | 否 | 中等 | 经济/中等/豪华 |
| 交通方式 | 单选 | 否 | 公共交通 | 公共交通/自驾/打车/混合 |
| 住宿类型 | 单选 | 否 | 经济型酒店 | 经济型酒店/舒适型酒店/豪华型酒店/民宿 |

**输出内容**：
- 每日行程（景点名称、地址、游览时间、描述、图片）
- 餐饮安排（早/中/晚三餐推荐）
- 住宿推荐
- 天气预报
- 预算明细
- 总体旅行建议

**验收标准**：
- [ ] 输入有效参数后 2 分钟内返回结果
- [ ] 每日至少包含 2-3 个景点
- [ ] 预算明细四舍五入到整数
- [ ] 景点信息包含真实坐标
- [ ] 异常输入有明确错误提示

#### F2: 地图可视化

**功能描述**：
- 在结果页嵌入高德地图
- 自动以第一个景点为中心定位
- 每个景点标注 Marker
- 显示景点之间的连线

**验收标准**：
- [ ] 地图正确加载
- [ ] 所有景点有 Marker 标注
- [ ] 点击 Marker 显示景点名称
- [ ] 地图缩放级别合适

#### F3: 预算明细

**功能描述**：
- 展示四项费用：门票、酒店、餐饮、交通
- 底部展示总费用
- 使用卡片/统计组件展示

**计算规则**：
- 门票：所有景点 ticket_price 之和
- 酒店：hotel.estimated_cost × 住宿晚数
- 餐饮：所有 meals.estimated_cost 之和
- 交通：人均估算（公共交通 30元/天，自驾 100元/天，打车 80元/天）

**验收标准**：
- [ ] 四项费用正确计算
- [ ] 总费用等于各项之和
- [ ] 费用为整数

#### F4: 天气信息

**功能描述**：展示行程期间每日天气，包括天气状况、温度范围。

**验收标准**：
- [ ] 天气日期与行程日期一致
- [ ] 温度为纯数字展示
- [ ] 天气查询失败时有降级处理

#### F5: 行程编辑

**功能描述**：
- 进入编辑/只读两种模式切换
- 编辑模式：上移、下移、删除景点
- 保存后刷新地图
- 取消恢复原始数据

**验收标准**：
- [ ] 编辑操作后行程更新
- [ ] 取消后完全恢复
- [ ] 地图同步更新

#### F6-7: 导出功能

**功能描述**：支持导出为 PDF 和图片文件

**验收标准**：
- [ ] 导出 PDF 格式正确
- [ ] 导出图片清晰
- [ ] 处理地图区域兼容性问题

## 4. 非功能需求

| 编号 | 类型 | 要求 |
|------|------|------|
| NF1 | 性能 | API 响应时间 < 120 秒（考虑 LLM 调用） |
| NF2 | 可用性 | 前端界面支持桌面浏览器（Chrome/Edge/Firefox） |
| NF3 | 可维护性 | 前后端分离，接口符合 RESTful 规范 |
| NF4 | 安全性 | API Key 仅存储在服务端环境变量中，不暴露给前端 |
| NF5 | 国际化 | 首版支持中文，后续可扩展多语言 |
| NF6 | 错误处理 | 外部 API 调用失败时有友好的降级提示 |

## 5. 用户故事

### US1: 快速生成旅行计划
> 作为一个自由行新手，我希望只需要输入目的地和时间，就能获得一份完整的旅行计划，这样我就不需要在多个网站之间来回查找信息。

### US2: 个性化偏好
> 作为一个历史文化爱好者，我希望能根据我的兴趣偏好生成行程，而不是每次都推荐相同的热门景点。

### US3: 预算控制
> 作为一个预算有限的自由行游客，我希望在生成行程时能看到费用预估，这样我可以提前合理安排花销。

### US4: 地图感知
> 作为一个不熟悉目的地城市的游客，我希望能在地图上看到所有景点的位置，这样我可以了解景点之间的距离和路线。

### US5: 行程微调
> 作为旅行计划者，我希望能对生成的行程进行微调，比如更换景点顺序或删除不感兴趣的景点。

### US6: 离线使用
> 作为一个旅行者，我希望能将行程导出为 PDF 或图片保存到手机，以便在没有网络的情况下随时查看。
